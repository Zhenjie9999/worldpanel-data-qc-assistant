# Worldpanel Data QC Assistant - 内网 HTTP LLM 接入设计

日期：2026-06-02

## 1. 目标

在现有 Windows 本地版中接入公司可访问的 Chat Completions 模型接口，用于补充业务逻辑检查。

模型能力用于识别和解释：

- `Share`、`Contribution` 等占比指标加总异常。
- 同组产品或品牌中的显著离群值，例如多数产品均价约为 `60`，某产品均价为 `1000`。
- 指标之间的逻辑冲突，例如均价、金额、销量关系不合理。
- 规则难以穷举，但结合指标名、品类、时间和上下文可以判断的业务疑点。

模型不替代本地规则引擎，不自动修改源文件，不自动关闭疑点。

## 2. 方案选择

采用混合 QC：

```text
Excel / PPTX / PDF 解析
-> 本地确定性规则与统计规则筛选
-> 生成最小化结构化片段
-> 内网 HTTP LLM 补充业务判断
-> 合并为待用户逐条确认的疑点
```

不采用“把完整文件直接发送给模型”的方案。原因：

- HTTP 没有 TLS 加密，只适合可信内网测试。
- 完整文件包含不必要的数据，风险和 token 成本更高。
- `Share` 加总等确定性问题由程序计算更稳定。
- 模型更适合解释异常、判断语义和提示人工复核方向。

## 3. 配置与密钥

新增本地模型设置页，包含：

- API 地址
- 模型名
- API 密钥
- 请求超时
- 启用 / 停用 LLM 逻辑复核
- 测试连接按钮

默认测试配置：

```text
API 地址：http://218.241.201.171:28180/jdgpt/v1/chat/completions
模型名：gpt-5.4
```

API 密钥不写入代码、文档、日志或导出报告。密钥保存在本机配置中，并使用 Windows DPAPI 加密。

当接口地址为 `http://` 时，界面固定显示：

```text
仅限可信内网测试。HTTP 未启用传输加密，请勿在公网环境使用。
```

未来接口切换为 HTTPS 时，不需要修改 QC 业务逻辑。

## 4. 数据最小化

默认不向模型发送：

- 完整 Excel、PPTX 或 PDF 文件
- 文件二进制内容
- 整个工作簿
- 全部 Slides 文本
- API 密钥

模型只接收：

- 项目名
- 文件名
- sheet、页码或单元格区域
- 指标名、分类名、时间范围和单位
- 一小段候选数据
- 本地规则计算结果
- 要模型回答的明确问题

每次调用记录：

- 使用的模型和接口主机
- 文件、sheet、页码或区域
- 调用时间
- 调用状态
- 发送的数据行数
- 模型返回的置信度

日志不记录 API 密钥，不保存完整提示词。

## 5. 本地规则

第一版新增以下确定性检查：

### 5.1 占比加总

- 识别指标名包含 `share`、`contribution`、`占比`、`份额` 的表格区域。
- 按相同 period、scope 和层级分组。
- 检查加总是否接近 `100%` 或 `1.0`。
- 容忍误差默认 `±0.5` 个百分点。
- 超出容忍范围时生成疑点，并将结构化片段发送给 LLM 解释。

### 5.2 离群值

- 在同一指标、period、scope 和层级中比较分类值。
- 使用中位数和 MAD / IQR 等稳健统计方法筛选异常值。
- 避免把全体业务增长误判成异常。
- 将离群值、同组基准值和上下文发送给 LLM 判断业务合理性。

### 5.3 指标关系

- 当上下文可以识别 `Price`、`Spend`、`Volume` 时，检查 `Price = Spend / Volume`。
- 当上下文不足时，不强行计算，交给 LLM 标记为需要复核。

## 6. LLM 输出

要求模型返回结构化 JSON：

```json
{
  "issues": [
    {
      "severity": "High | Medium | Low",
      "title": "简短标题",
      "description": "为什么可疑",
      "evidence": "关键数字和对照值",
      "recommendation": "建议复核什么",
      "confidence": 0.0
    }
  ]
}
```

LLM 疑点写入现有 `Current File QC`，规则来源标记为：

```text
llm_logic_review
```

用户仍然逐条选择：

```text
Pending
Confirmed error
Fixed
Confirmed OK
Needs review
```

## 7. 失败处理

- 模型接口不可达：本地规则继续运行，记录 `connection_failed`。
- 返回非 JSON：记录 `invalid_response`，不阻塞整个 QC。
- 超时：记录 `timeout`，提示用户可以稍后重试。
- HTTP 模式：始终显示风险警告。
- 模型没有发现问题：记录成功调用，不新增疑点。

## 8. 界面变化

新增：

- `LLM Settings` 设置入口。
- 内网 HTTP 风险提示。
- 测试连接结果。
- 新建 QC 时的 `Enable LLM logic review` 开关。
- `AI Logs` 中的模型调用记录。
- 疑点列表中的 `LLM logic review` 来源。

## 9. 验收标准

- 使用纯合成请求验证接口可连接。
- 密钥未出现在仓库、日志、报告或网页响应中。
- `Share` 加总为 `90%` 的合成表格会生成疑点。
- 同组均价为 `55、60、62、1000` 的合成表格会生成离群值疑点。
- 模型不可达时，本地检查仍能完成。
- LLM 返回非 JSON 时，QC 不崩溃。
- 真实文件测试时，只发送筛选后的结构化片段。

## 10. 已知边界

- 当前 HTTP 接口只作为可信内网测试能力。
- 外网或公司网站正式部署前，应优先配置 HTTPS、访问控制和密钥轮换。
- 模型可能产生误判，所有结果必须由用户确认。
- 泛化业务规则需要通过真实项目持续补充。
