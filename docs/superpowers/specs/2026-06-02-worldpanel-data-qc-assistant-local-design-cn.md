# Worldpanel Data QC Assistant - Windows 本地测试版产品设计

日期：2026-06-02

## 1. 产品定位

Worldpanel Data QC Assistant 是一个面向 Worldpanel 分析、项目管理和交付团队的数据质检产品。

产品目标包括：

1. 防止包含错误的数据文件或报告交付给客户。
2. 提高分析师检查 Powerview 数据产出的效率。
3. 建立可追溯、可复用的团队 QC 工作流程。

第一阶段不连接 Powerview，也不负责从 Powerview 导数。系统从用户已经准备好的 Excel、PowerPoint 和 PDF 文件开始工作。

```text
用户已有 Excel / PPTX / PDF
-> 上传文件
-> 自动提取和识别数据
-> 分别执行文件内部 QC 和跨文件 QC
-> 用户逐条确认疑点
-> 生成可留档的 QC 结果
```

第一阶段先开发 Windows 本地测试版。规则、准确率和使用流程经过真实项目验证后，再迁移到公司网站，实现集中保存、共享和团队级管理。

## 2. 第一阶段产品形态

采用 Windows 本地 Web App。

用户双击启动程序后，系统在本机运行服务，并自动打开浏览器中的本地网页。用户不需要安装开发环境。

选择这一方案的原因：

- 适合使用 Office 文件的日常工作环境。
- 本地测试期间能够快速验证真实文件。
- 界面、规则引擎和数据结构可以迁移到未来公司网站。
- 识别服务可以替换：测试版可以使用外部 AI，正式版可根据公司安全策略切换为公司批准服务或本地模型。

## 3. 用户和身份识别

本地测试版不接入公司账号系统。

用户首次打开时填写：

- 姓名
- 公司邮箱

系统保存在本机，并用于记录：

- 谁发起 QC
- 谁新增或修改项目规则
- 谁确认疑点
- 谁修改疑点状态
- 谁完成最终 QC 确认

未来迁移到公司网站时，可以将轻量身份识别替换为正式账号体系。

## 4. 支持的文件格式

| 类型 | 格式 | 处理方式 |
| --- | --- | --- |
| Excel | `.xlsx` | 直接读取数值、公式、格式、sheet、隐藏区域和表格结构 |
| Excel 旧格式 | `.xls` | 本地转换后读取，并记录转换状态和潜在格式损失风险 |
| Slides | `.pptx` | 读取文本、表格、可编辑图表、图片和复杂组合图形 |
| 最终交付文件 | `.pdf` | 提取文本和图片，并结合 OCR / AI 检查客户最终可见内容 |

第一阶段不支持旧版 `.ppt`。遇到 `.ppt` 时，提示用户另存为 `.pptx` 后再上传。

## 5. 项目和单次 QC

用户先选择已有项目，或新建项目。

系统可以根据文件名建议项目，但最终由用户确认。

项目示例：

```text
Zespri PanelVoice
Coca-Cola Walmart Review
Fresh Panel Credential
```

本地测试版一次只处理一个项目中的一组文件，不支持批量项目队列。

每个项目保存：

- 项目信息
- 项目级自定义规则
- 历史 QC 运行记录
- 上传文件副本或索引
- 文件之间的映射关系
- 用户确认过的数字来源
- 版本对比关系
- 疑点和处理状态
- 导出的报告

## 6. 核心使用流程

```text
首次打开：填写姓名和公司邮箱
-> 选择已有项目 / 新建项目
-> 上传一组 Excel、PPTX、PDF 文件
-> 系统识别文件类型
-> 系统推荐是否与历史版本对比
-> 用户确认版本关系
-> 可选：设置 PPTX/PDF 页码与 Excel 文件或 sheet 的映射约束
-> 自动检查
-> 查看当前文件 QC
-> 查看与上一版差异
-> 用户逐条确认疑点状态
-> 系统建议整体 QC 状态
-> 用户确认完成 QC
-> 导出 Excel 明细和 PDF 摘要
```

## 7. 文件内部检查

### 7.1 Excel QC

检查 `.xlsx` 和转换后的 `.xls`：

- 数值、公式、sheet、表格区域和隐藏行列
- 空值、错误值和异常 0
- Total、Subtotal、Share 和 Contribution 加总关系
- 同比、环比、百分比、单位、小数位和正负号
- 核心 KPI 逻辑关系，例如 `Price = Spend / Volume`
- 同一文件前后 period、scope 和指标名称是否一致
- 公式被手动覆盖、同一列公式结构异常
- 多 Excel 之间的口径和重复数字一致性

### 7.2 PPTX QC

检查客户或项目团队能看到的内容：

- 文本框
- 表格
- 可编辑 chart series
- 图片化图表
- 截图
- 复杂组合图形
- 标题、结论、source、base、period 和 footnote
- 同页加总和跨页 KPI 一致性
- 标题结论与图表指标是否冲突
- Placeholder、乱码和未完成页面

### 7.3 PDF QC

将 PDF 作为最终客户视角进行检查：

- 提取页面文本和可见数字
- OCR 识别图片化页面
- 检查 source、footnote、period 和跨页一致性
- 与 PPTX 对照，发现导出后的漏页、漏内容或版本错误
- 与 Excel 对照，发现最终交付数字与 source 数据不一致

## 8. 全页面覆盖和人工复核

复杂图片图表、组合图形和 PDF 页面不能被静默跳过。

每个 PPTX / PDF 页面都需要展示：

```text
自动识别覆盖率
已识别数字数量
低置信度数字数量
无法识别区域数量
需要人工确认的区域
```

用户可以打开页面预览，查看 OCR / AI 标记的区域，并确认识别结果。

系统不承诺复杂图片图表 100% 自动识别。无法可靠识别的内容必须进入人工复核清单。

## 9. 外部 AI 和 OCR

为了提高测试版效率和识别覆盖率，外部 AI / OCR 默认开启。

外部服务可用于：

- 识别 PPTX 中图片化图表的数字和文字
- 识别 PDF 页面中的数字、source 和 footnote
- 理解复杂页面结构
- 标记低置信度区域

界面必须明确提示可能有文件页面发送到外部服务，并允许用户关闭。

系统记录：

```text
使用了哪些服务
发送了哪些文件和页码
执行时间
操作者
识别结果置信度
```

正式上线前，需要根据公司安全策略决定：

- 是否允许继续使用外部服务
- 允许使用哪些公司批准的服务
- 是否需要改为本地 OCR 或公司内部模型
- 是否需要脱敏

## 10. 跨文件检查

当用户上传 Excel、PPTX 和 PDF 时，系统分别执行：

```text
Excel 内部 QC
PPTX 内部 QC
PDF 内部 QC
PPTX vs Excel
PDF vs PPTX
PDF vs Excel
```

### 10.1 自动匹配依据

系统根据以下信息推荐数字来源：

- 数值及四舍五入差异
- 百分比与小数等价关系，例如 `40.2%` 与 `0.402`
- 单位转换，例如 raw value、thousand、million
- 指标名称
- 品牌、品类、时间和区域
- Excel 文件名和 sheet 名
- PPTX/PDF 页码周边文字
- 项目内历史确认记录

### 10.2 多候选来源

如果系统发现多个可能的 Excel 来源：

```text
自动选择最高置信度候选
-> 展示其他候选来源
-> 用户可以修正
-> 项目内复用确认记录
```

### 10.3 手动映射约束

复杂项目允许用户提前限制来源范围：

```text
PPTX 第 6-10 页 -> 0527.xlsx / Summary sheet
PPTX 第 17 页 -> gifting.xlsx
PDF 第 8 页 -> working_deck.pptx 第 8 页
```

简单项目无需设置，直接使用自动匹配。

## 11. 版本对比

当前文件 QC 和历史版本变化必须分开显示：

```text
Tab 1：Current File QC
Tab 2：Changes vs Previous Version
```

### 11.1 自动建议对比条件

系统仅在以下条件同时满足时建议进行版本对比：

```text
文件类型相同
文件名相似度 >= 90%
```

计算文件名相似度时，忽略：

- 日期，例如 `0527`、`20260527`
- 版本词，例如 `V1`、`V2`、`Final`、`Updated`
- 空格、下划线和短横线

达到 90% 时，系统只做建议，最终仍由用户确认。

相似度不足时，用户仍然可以手动选择历史文件进行对比。

用户也可以明确选择：

```text
本次不做版本对比
```

### 11.2 版本差异内容

Changes vs Previous Version 单独列出：

- 数字变化，例如 `16.2% -> 16.6%`
- 文本变化，例如标题、source 和 footnote 修改
- 新增或删除 slide、sheet 和 chart
- 新增疑点
- 已修正疑点
- 持续存在的问题

## 12. 疑点处理和 QC 闭环

系统负责发现疑点、展示证据和提供建议，但不自动修改原文件。

每个疑点支持状态：

```text
待处理
确认有误
已修正
确认无误
需要复核
```

用户可以填写备注。

系统根据问题状态给出整体建议：

| 状态 | 含义 |
| --- | --- |
| `Not Ready` | 仍有未关闭的 High-risk 问题 |
| `Needs Review` | 没有未关闭 High-risk 问题，但仍有低置信度区域或待人工复核项 |
| `Ready for Delivery` | High-risk 问题已关闭，人工复核区域已确认 |

最终是否完成 QC，由用户显式确认，并留下姓名、时间和备注。

## 13. 规则体系

本地测试版采用：

```text
内置 Worldpanel 标准规则
+ 项目级自定义规则
```

第一阶段不做个人规则、团队管理员权限和复杂脚本规则。

项目级自定义规则支持以下可配置类型：

- 必须包含指定文字
- 禁止包含指定文字
- 数值范围
- 百分比加总容忍值
- 必须存在 source、base、period 或 footnote
- 指定页码、文件或 sheet 的映射约束
- 指标小数位规范
- 指标单位规范

示例：

```text
项目：Zespri PanelVoice
规则：所有相关页面必须包含 Imported Kiwi to proxy Zespri
范围：PPTX / PDF footnote
严重程度：High
```

## 15. 本地存储

每个项目在本机保存：

- 项目信息
- 用户信息
- 上传文件副本或索引
- QC 运行记录
- 版本关系
- 抽取的数据
- OCR / AI 调用日志
- 疑点和用户处理状态
- 用户确认记录
- 项目规则
- 导出的 Excel / PDF 报告

未来迁移到公司网站时，保留相同数据结构，将本地存储替换为集中式数据库和文件存储。

## 16. 报告输出

本地测试版提供：

1. 本地网页中的 QC 查看和处理。
2. Excel 明细导出。
3. PDF 摘要导出。

Excel 明细建议包含：

- 当前文件 QC
- 与上一版差异
- 页面覆盖率
- 人工复核项
- 疑点处理状态
- 用户备注
- QC 确认记录

PDF 摘要建议包含：

- 项目和文件信息
- QC 整体状态
- High / Medium / Low 数量
- 未关闭高风险问题
- 低置信度和人工复核情况
- 与上一版的主要变化
- QC 确认人和时间

第一版不自动写回 Excel、PPTX 或 PDF，不自动添加 PPT 批注。

## 17. 第一阶段验收标准

Windows 本地测试版需要做到：

1. Windows 双击启动，本地浏览器自动打开。
2. 支持 `.xlsx`、`.xls`、`.pptx` 和 `.pdf`。
3. 一次处理一个项目的一组文件。
4. 每页 PPTX / PDF 都有覆盖率和人工复核提示。
5. 分别检查 Excel、PPTX 和 PDF 内部问题。
6. 执行 Excel-PPTX-PDF 的交叉检查。
7. 按文件名相似度建议版本对比，并由用户确认。
8. 分开显示当前疑点和版本差异。
9. 用户可以逐条更新疑点状态并填写备注。
10. 保存项目规则、历史记录和确认痕迹。
11. 导出 Excel 明细和 PDF 摘要。
12. 高风险问题未关闭时，不建议标记为 `Ready for Delivery`。
13. 测试版默认开启外部 AI / OCR，同时提供关闭入口和调用日志。

## 18. 明确不做

第一阶段不做：

- 连接 Powerview
- 从 Powerview 导数
- 支持旧版 `.ppt`
- 批量项目队列
- 自动修改 Excel、PPTX 或 PDF 原文件
- 自动写回 PPT 批注
- 个人规则和团队管理员权限
- 复杂脚本规则
- 承诺复杂图片图表 100% 自动识别

## 19. 后续公司网站版本

本地测试稳定后，迁移到公司网站。

网站版重点增加：

- 正式公司账号登录
- 集中式数据库和文件存储
- 团队共享项目
- 团队标准规则管理
- 管理员角色和权限
- QC 记录集中保存
- 团队 QC Dashboard
- 批量任务和任务队列
- 公司批准的 AI / OCR 服务
- 文件安全、脱敏和审计策略
