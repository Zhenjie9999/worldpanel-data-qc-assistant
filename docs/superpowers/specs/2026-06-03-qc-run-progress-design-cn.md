# QC 检查进度反馈设计

## 目标

用户上传文件后立即看到检查进度，不再面对长时间静止页面。界面保持简单，只提供阶段、进度条、预计剩余时间和失败提示。

## 用户体验

点击 `Run QC` 后，系统立即创建运行记录并打开运行页面。运行页面显示：

- 当前阶段，例如 `AI data review`
- 进度条和百分比
- 当前进展，例如 `Reviewing Slides page 6 / 17`
- 预计剩余时间，例如 `About 2 minutes remaining`

检查完成后，页面自动切换到完整报告。检查失败时，页面显示失败阶段和简短错误信息。

## 后台处理

现有同步检查改为后台线程执行。创建运行接口只负责保存上传文件、创建运行记录并立即返回。后台任务逐步更新数据库中的进度字段。

阶段：

1. `Queued`
2. `Reading files`
3. `Running local rules`
4. `AI data review`
5. `Slides visual review`
6. `Preparing report`
7. `Completed`

预计剩余时间根据已经耗时、当前百分比和待审核 Slides 页数估算。它是近似值，不作为精确承诺。

## 页面刷新

运行页面每 2 秒获取一次当前状态。运行完成或失败后停止轮询。用户刷新页面后仍然能够看到后台任务的当前进度。

## 失败处理

后台任务捕获异常并将运行状态更新为 `Failed`。页面显示失败阶段和错误摘要，保留上传文件和运行记录，方便后续排查。

## 范围限制

本次不增加复杂任务中心、不增加暂停和取消、不展示技术日志、不调整现有报告结构。
